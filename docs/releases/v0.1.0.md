# Release Notes: v0.1.0

Release date: 2026-02-27

## Summary

`v0.1.0` is the first release-ready baseline of **author-collector**.
It delivers a compliant, evidence-first pipeline with:

- strict schema contract
- safe fetch layer (robots/rate/security)
- deterministic parsing/extraction
- idempotent storage/versioning/rollback
- three working connectors (RSS, HTML author page, arXiv)
- manual-only identity review/apply loop

## Delivered Scope (M0-M5)

- M0: core models + contract tests + compliance defaults
- M1: fetcher compliance foundation
- M2: parser + extractor with evidence linkage
- M3: storage + dedup + versioning + export + rollback
- M4: RSS / HTML author page / arXiv connectors
- M5: scoring + review queue + review apply (no auto-merge)

## Reliability and Operability in This Release

- structured JSON logs with `run_id`
- fail-fast export schema validation
- rollback-by-run support
- CI gates:
  - `ruff check .`
  - `pytest` with coverage threshold `>= 80%`
- daily canary workflow and artifacted report

## Known Boundaries

- no automatic identity merge
- no full body storage
- v0 migration policy is additive-only
- branch-protection automation requires external repo admin token

## Upgrade / Verification Checklist

1. Pull latest `main`.
2. Run `ruff check .`.
3. Run `pytest -q`.
4. Run a smoke sync:
   - `author-collector sync --source-id rss:example_feed --seed tests/fixtures/rss/example.xml --run-id smoke-v010`
5. Run export validation:
   - `author-collector export --output /tmp/smoke-v010.jsonl --run-id smoke-export-v010`
